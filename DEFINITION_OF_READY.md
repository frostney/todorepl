# Definition Of Ready

## Executive Summary

- Work starts from a GitHub issue or confirmed mini-spec.
- Scope, acceptance criteria, and non-goals must be clear before implementation.
- Source evidence must be checked before planning.
- New product-shaping decisions run through `grill-with-docs`.

## Ready Checklist

An issue or mini-spec is ready for implementation when:

- The user-facing outcome is stated plainly.
- Acceptance criteria describe observable behavior.
- Relevant source files, docs, and skills have been read.
- Dependencies and constraints are known, including storage, CLI, and agent-output implications.
- Any conflict with `VISION.md` has been surfaced and resolved.
- Any product-shaping ambiguity has gone through `grill-with-docs`.
